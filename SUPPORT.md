# Support

Este archivo explica como pedir ayuda de forma útil para el equipo.

## Cuando abrir una issue

Abre una issue cuando necesites:

- reportar un bug reproducible;
- pedir una mejora pequeña o una tarea técnica;
- aclarar un comportamiento del sistema;
- documentar una incidencia de despliegue o ambiente.

## Qué incluir siempre

- objetivo o problema en una frase;
- pasos para reproducirlo;
- comportamiento esperado;
- comportamiento real;
- capturas o logs si aportan contexto;
- rama, commit o ambiente donde ocurrio.

## Como pedir ayuda bien

1. Usa la plantilla correcta de issue.
2. Describe el contexto mínimo necesario.
3. Indica si el problema bloquea el trabajo o si hay una solución provisional.
4. Si el problema ya fue investigado, deja lo que probaste para no repetir esfuerzos.

## Casos urgentes

Si una incidencia afecta disponibilidad, seguridad o despliegue, marca el impacto claramente y enlaza la evidencia técnica disponible. El objetivo es que otra persona pueda retomar el caso rápido sin reconstruir toda la historia.
