# Support

Este archivo explica como pedir ayuda de forma util para el equipo.

## Cuando abrir una issue

Abre una issue cuando necesites:

- reportar un bug reproducible;
- pedir una mejora pequena o una tarea tecnica;
- aclarar un comportamiento del sistema;
- documentar una incidencia de despliegue o ambiente.

## Que incluir siempre

- objetivo o problema en una frase;
- pasos para reproducirlo;
- comportamiento esperado;
- comportamiento real;
- capturas o logs si aportan contexto;
- rama, commit o ambiente donde ocurrio.

## Como pedir ayuda bien

1. Usa la plantilla correcta de issue.
2. Describe el contexto minimo necesario.
3. Indica si el problema bloquea el trabajo o si hay una solucion provisional.
4. Si el problema ya fue investigado, deja lo que probaste para no repetir esfuerzos.

## Casos urgentes

Si una incidencia afecta disponibilidad, seguridad o despliegue, marca el impacto claramente y enlaza la evidencia tecnica disponible. El objetivo es que otra persona pueda retomar el caso rapido sin reconstruir toda la historia.
